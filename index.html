Built like a complete product

Chapa goes beyond a prompt box: it includes auth, persistence, voice, offline fallback, settings, widget support, and a backend that owns context.

00 / Design

Designed first in Figma

The full app interface was designed from scratch in Figma before implementation, then translated into Compose with the same product identity.

01 / Voice loop

Tap, speak, reply

Audio recording, waveform rendering, automatic silence detection, transcription, and assistant responses in one conversational flow.

02 / Context

Backend-owned memory

The backend stores conversation history and sends only the relevant context window to the model, instead of pushing full history from Android every time.

03 / Offline

Text fallback on device

When connectivity drops, Chapa disables audio and uses a local LiteRT-LM model for occasional text replies while Room keeps chat history visible.

04 / Trust

Encrypted persistence

Conversation titles, message text, and reply previews are encrypted by the backend before they reach Firestore.

05 / Interaction

Feels like messaging

Pin chats, rename, delete, reference messages, play voice notes, listen to TTS, and browse history from a pinned drawer footer.

06 / Delivery

Ready to review

Clean architecture, Hilt, tests, Room cache, Firebase Auth, FastAPI, Swagger docs, and setup instructions for local and Vercel deployments.

Security posture

No provider keys in the APK

Firebase ID tokens authenticate every backend request.
OpenAI keys and encryption keys live only in the backend environment.
Firestore stores encrypted sensitive text when `CHAPA_ENCRYPTION_KEY` is configured.
Legacy plaintext documents remain readable for migration compatibility.
